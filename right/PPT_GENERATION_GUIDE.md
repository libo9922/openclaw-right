## 二、配色方案

### 推荐：深蓝灰 + 暖米色 + 古铜金（学术/历史/商务）

| 角色 | 色值 | 用途 |
|------|------|------|
| 深蓝 NAVY | `#1B2A4A` | 主背景、标题栏 |
| 更深 DEEP_NAVY | `#0A1225` | 封面背景 |
| 暖米 CREAM | `#F5F0E8` | 浅色文字、浅色卡片底 |
| 羊皮纸 PARCHMENT | `#E8DED0` | 浅色页面背景 |
| 金色 GOLD | `#C49A6C` | 装饰条、章节标签、强调 |
| 深红 CRIMSON | `#A03040` | 警示、重点结论 |
| 灰 GRAY | `#6B7B8D` | 次要文字 |

### 其它可用方案

- **科技蓝**: `#0D1B2A` + `#1B98E0` + `#E0E1DD`
- **中国红**: `#8B0000` + `#F5F0E8` + `#D4A574`
- **极简黑白**: `#1A1A1A` + `#FFFFFF` + `#FF6B35`（点缀橙）

## 三、布局套路

| 页面类型 | 结构 |
|----------|------|
| **封面** | 深色满背景 + 装饰边框 + 大标题 + 副标题 + 作者 |
| **内容页** | 深色标题栏（1.3寸高） + 双栏卡片 |
| **对比页** | 标题栏 + 表格（矩形拼） |
| **时间线** | 标题栏 + 逐行色块（左金条 + 日期 + 标题 + 描述） |
| **结语** | 同封面风格 + 引文居中 |

### 核心组件

- **rect** — 矩形色块（背景、装饰条、卡片底）
- **textbox** — 文字（通过 `add_paragraph()` 追加多段）
- **oval** — 序号圆圈
- **diamond** — 装饰菱形

## 四、沉淀经验

### 做什么

- **先写大纲再写代码**：确定几页、每页什么结构
- **用 layout[6]（空白）**：最灵活，不受默认占位符限制
- **卡片式布局**：rect 打底 + textbox 填内容，比表格灵活
- **双栏对比**：左右各一个 card，视觉冲击力强
- **装饰细节**：顶/底金色细条、左侧竖条、菱形点缀，成本低效果好

### 别做什么

- **别用默认 layout**：占位符位置固定，很难调出好看的效果
- **别堆太多字**：每页 6-8 行要点，多余的内容拆到下一页
- **别用太多种字体**：一个主字体 + 一个强调字体就够了
- **别忘记 `line.fill.background()`**：默认形状有蓝边框，很丑

### 常见坑

| 坑 | 解决 |
|----|------|
| 形状默认有蓝色边框 | `shape.line.fill.background()` |
| 文字不换行 | `text_frame.word_wrap = True` |
| 居中不生效 | 要在 `paragraph` 上设 `alignment`，不是 `text_frame` |
| 中文字体显示方块 | 确保字体名正确：`Microsoft YaHei`、`SimHei`、`SimSun` |
| 段落间距太大/太小 | `paragraph.space_before = Pt(6)` |
