## 四、能力迁移要点

### 🔑 核心原则

1. **先诊断再行动** — 读取关键文件状态，不要假设前序会话已完成
2. **工具组合** — `exec` 做 shell，`write`/`edit` 做文件，`read` 做信息获取
3. **记忆持久化** — 重要操作写 `memory/YYYY-MM-DD.md`，长期信息写 `MEMORY.md`
4. **版本控制** — 文件变更后 `git add . && git commit`
5. **子会话并行** — 独立任务用 `sessions_spawn`，不阻塞主会话
